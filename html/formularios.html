Formulários

Quando queremos preencher uma ficha ou captar uma resposta do usuário podemos fazer isso utilizando de várias formas com um formulário

Tag form

A tag form é reponsavel por captar os dados preenchidos em um campo de pergunta e enviá-los para uma outra página onde esses dados poderão ser tratados

Quando formos enviar um formulário temos que pensar de qual forma queremos mandar os dados captados e para onde, podemos por exemplo querer mandar para a própria página, para isso colocamos um atributo "action" com o valor nulo "" na tag "form", ou se quisermos podemos colocar para outras páginas para isso mude o valor do action com o caminho de arquivo ou com a url

Também é preciso informar qual o método do envio se vai ser por GET ou POST, o método GET faz com que o envio do formulário apareça no caminho da url as informações (cuidado com o GET, pois; dados sigilosos não devem ser passados por ele, a url possui uma limitação de caracteres logo se tiver muita informação sendo passada os dados podem se perder), já o método POST passa somente internamente os dados, não sendo visível suas informações

Após criar uma tag datalist com um id e dentro elementos filhos do tipo option, é preciso definir no campo input o atributo list passando o id do datalist que você quer usar

Tag input

Campos de perguntas que podem ser preenchidos

O tipo checkbox representa uma caixa que pode ser selecionada, se quiser deixar a caixa por padrão já selecionada use o atributo checked

Marque a seguir os esportes que você já jogou:

O tipo color serve para colocar uma cor o value é passado como código hex, se o campo value não for definido a cor padrão é preto #000000

O tipo date permite inserir uma data e seu valor deve ser passado como: ano-mês-dia, por exemplo 1998-12-30

O tipo datetime-local permite inserir uma data juntamente do horário e seu value é informado como: ano-mês-diaThora:minutos, por exemplo: 1998-12-30T22:59

O tipo email permite enviar o nome do seu email, como existem algumas regras para formar um email como a presença do @ essa tag surgiu

O tipo file permite passar um arquivo, para passar vários arquivos utilize o atributo multiple, é possível adicionar o atributo accept para informar quais são os únicos tipos de arquivos permitidos (apenas imagens: image/*, apenas audio/*, apenas video/*, apenas imagens e videos: image/*,video/*, apenas de extensão webp: .webp)

O tipo hidden serve para definir um value qualquer, utilizado para quando por exemplo queremos fazer uma tag que o form não lê como possível de ser passado, imagine que exista uma div e ao selecionar ela por script mudamos o valor do input hidden, assim quando enviarmos o formulário o que foi selecionado na div é passado pelo hidden

O tipo number permite apenas utilizar números e expressões matemáticas

O tipo password realiza o mesmo papel que o text, mas quando for preenchido aparece visualmente para o usuário como um bando de bolinhas

O tipo radio permite assim como o checkbox selecionar uma opção só que essa opção possui uma associação a uma pergunta podendo fazer esquemas como por exemplo uma pergunta de prova com 5 alternativas.

Para associar os campos de resposta, todos devem possuir o mesmo name, lembrando que assim que for marcado não poderá ser desmarcado a menos que faça um script passando o atributo checked como false ou um botão reset

Tag label

Cria um título para um campo de resposta, a dúvida ou texto relacionado ao campo de resposta poderá ser clicado para redirecionar o usuário a preencher com uma resposta

Tag select, optgroup, option

Cria um campo de resposta com uma lista de opções

Parâmetro - multiple - nos permite selecionar mais de um item das opções, porém só clicando não tem efeito, precisa segurar o CTRL e então clicar com o botão esquerdo para funcionar.

Atributo - size - permite visualizar uma quantidade de linhas das opções

Parâmetro - selected no option permite já começar selecionável o item

Dentro do select podemos agrupar os tipos de opções, ao agrupalos podemos colocar como se fosse um título não selecionável dentro do select

Tag textarea

Cria um grande campo de resposta para captar textos mais desenvolvidos
