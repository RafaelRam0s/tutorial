Regras de um bom site

  1. Informe o formato que ele trabalha, no caso de html5 !DOCTYPE

  2. Informe o idioma da sua página, o sentido de leitura, o zoom com o qual a sua página aparece em dispositivos menores e um ícone

  3. Defina para cada página do seu site um título, um meta description, keywords e author únicos

  4. Referências de arquivo css no head e de javascript no footer

  5. Nos arquivos css e javascript deixe somente o que irá utilizar, evite código morto

  6. Garanta primeiro um texto padrão como sans-serif, serif ou monospace e depois o carregamento de uma webfont caso utilize outra família de fonte

  7. Faça uma página de política de privacidade e termos de uso

  8. Tenha um h1 em cada página

  9. Verifique se todas as páginas são responsivas

  10. Tenha uma paleta de cores agradável

  11. Não use documentos pesados para trabalhos leves (exemplo: imagem de 1400x1200 para ficar em um elemento de 400x300)

  12. Algumas vezes um elemento no html carrega somente depois que o arquivo é baixado, o que causa um deslocamento do layout html, evite isso definindo width e height

  13. Teste todas as funcionalidades do site

  14. Configure um arquivo robots.txt e um sitemap.xml na rota principal para leitores de tela

  15. Minifique seus arquivos

  16. Deixe elementos estáticos, como imagens de logo, arquivos css e js, em cache

  17. Tenha uma página de erro, caso alguém coloque uma url incorretamente é bom que seja redirecionada para a tela de erro do seu site, pois assim você pode manter a pessoa dentro do sistema, se deixar o padrão do navegador ela pode desistir de visitar suas páginas

  18. Se utilizar cookies, informe que você utilize para o usuário

  19. Se utilizar imagens cuidado com os direitos de imagem

Como publicar seu site:

  1. Ter uma pasta no seu computador para os códigos
  2. Fazer um arquivo html (veja no W3schools)
  3. Ter uma linguagem back-end para servidor ("Asp.Net Core MVC" ou "Node.js")
  4. Ter um editor de código (recomendo Visual Studio Code)
  5. Comprar um nome de domínio (comprar a url basicamente, por exemplo existe o site www.exemplo.com essa parte "exemplo.com" é o nome de domínio e para obte-lo é preciso comprar em sites na internet, recomendo site https://registro.br/)
  6. Contratar um servidor de hospedagem (basicamente o servidor vai armazenar os arquivos que você tem e quando alguém acessar o seu domínio ele irá carregar os arquivos para o visitante)
  7. Comprar um certificado http para o site
  8. Ter software que fornece contato entre a sua máquina e o servidor (esses tipor de programas se chamam FTP, recomendo FileZilla)
  9. Configurar o FileZilla para conectar com o servidor
  10. Mandar arquivos do seu computador para servidor
